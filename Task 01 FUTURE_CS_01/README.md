# 🛡️ Web Vulnerability Assessment: Altoro Mutual

## 📝 Project Overview
This repository contains a professional security audit of the **Altoro Mutual** demo website. The goal was to identify common security weaknesses, classify their risks to the business, and suggest practical remediation steps.

This task was performed as part of a security consulting learning path, focusing on **professional reporting** and **ethical analysis**.

---

## 🌐 Website Tested
* **Target Name:** Altoro Mutual (Demo Bank)
* **URL:** [http://demo.testfire.net](http://demo.testfire.net)
* **Technology Stack:** Apache Tomcat/Coyote 1.1, JSP, Java

---

## 🎯 Scope & Ethics
This audit followed strict **ethical guidelines**:
* ✅ **Passive Analysis:** Only observed data sent from the server.
* ✅ **Read-only Testing:** No data was modified or deleted.
* ✅ **Public-facing Pages:** Only analyzed pages available to the public.
* ❌ **No Exploitation:** Identified vulnerabilities were documented but not "hacked" or exploited.
* ❌ **No Denial-of-Service:** Testing was non-intrusive to ensure site availability.

---

## 🛠️ Tools Used
* **Nmap** 🛰️ – Used for port scanning and service version detection.
* **OWASP ZAP** 🕵️‍♂️ – Used for automated passive scanning to identify SQLi, XSS, and configuration errors.
* **Browser DevTools** 🖥️ – Used for manual inspection of HTTP response headers and client-side issues.
* **Canva** 🎨 – Used to design the final professional Vulnerability Assessment Report.
